# AdService

REST API для сервиса размещения объявлений, разработанный на Django с использованием Django REST Framework.

## Возможности

- регистрация и авторизация пользователей;
- создание, просмотр, редактирование и удаление объявлений;
- загрузка изображений к объявлениям;
- поддержка API v1 и API v2;
- автоматическая документация Swagger и ReDoc.

## Используемые технологии

- Python
- Django
- Django REST Framework
- drf-spectacular
- SQLite
- Pillow

## Установка и запуск (Windows)

Клонируйте репозиторий:

```bash
git clone https://github.com/weahem/adservice
cd adservice
```

Создайте виртуальное окружение:

```bash
python -m venv .venv
```

Активируйте его:

```bash
.venv\Scripts\activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

Примените миграции:

```bash
python manage.py migrate
```

При необходимости создайте суперпользователя:

```bash
python manage.py createsuperuser
```

Запустите сервер разработки:

```bash
python manage.py runserver
```

После запуска проект будет доступен по адресу:

```
http://127.0.0.1:8000/
```

## Документация API

Для API v2 используется библиотека **drf-spectacular**, автоматически генерирующая документацию OpenAPI.

После запуска проекта доступны следующие страницы:

| Страница | Адрес |
|----------|-------|
| Swagger UI | `/api/schema/swagger-ui/` |
| ReDoc | `/api/schema/redoc/` |
| OpenAPI Schema | `/api/schema/` |

Swagger UI позволяет:

- просматривать все доступные эндпоинты;
- изучать параметры запросов и ответов;
- просматривать модели данных;
- выполнять запросы непосредственно из браузера;
- проверять работу API без использования Postman.

ReDoc предоставляет документацию в более удобном формате для ознакомления со структурой API и описанием моделей.

## Версии API

### API v1

Первая версия API реализована средствами Django без использования Django REST Framework.

Особенности:

- ручная обработка HTTP-запросов;
- формирование JSON-ответов средствами Django;
- сохранена для обеспечения обратной совместимости.

Для проверки работы API v1 в проекте предусмотрена отдельная страница тестирования:

```
http://127.0.0.1:8000/test_all_api.html
```

Страница содержит формы для выполнения запросов к основным методам API и позволяет протестировать работу сервиса непосредственно через браузер без использования сторонних инструментов.

### API v2

Вторая версия API реализована с использованием Django REST Framework.

Основные отличия:

- использование `ModelViewSet`;
- сериализация данных через DRF Serializer;
- маршрутизация с помощью `DefaultRouter`;
- автоматическая генерация OpenAPI-документации;
- поддержка Swagger UI и ReDoc;
- более простое расширение и сопровождение проекта.

Для проверки API v2 рекомендуется использовать Swagger UI, который предоставляет интерактивный интерфейс для выполнения запросов и просмотра ответов.

## Кастомная авторизация пользователя

Добавлена кастомная авторизация пользователя с помощью токена авторизации. Теперь для корректной проверки API v2 необходимо использовать токен авторизации пользователя.

Как получить токен:
1. Открыть сваггер
2. Выбрать метод `POST /api/v2/login/`
3. Ввести в request body формата `application/json` в формате
   ```JSON
   {
     "username": "<ваше имя пользователя>",
     "password": "<ваш пароль>"
   }
   ```
4. Нажать "execute"
5. Найти в response body строку формата
   ```JSON
   {
      "token": "<авторизационный токен>"
   }
   ```
6. Сверху справа нажать кнопку Authorize
7. Ввести в поле tokenAuth строку формата
   ```
   Token <авторизационный токен>
   ```
8. Нажать кнопку Authorize

# Асинхронная загрузка файла

Минималистичный сервер на базе фреймворка `aiohttp` для загрузки файлов без сохранения их на диск и без полной загрузки в память. Реализует один эндпоинт для определения размера переданного файла.
Запуск сервера происходит из папки t2 проекта командой
```bash
python main.py
```
Обращение к endpoint производится посредством postman по адресу `http://localhost:8080/api/upload/` с методом POST. Для отправки файла необходимо выбрать тело запроса `multipart/form-data`. В качестве ключа необходимо прописать file с типом `file` и в качестве значения указать любой файл.
Ответ от сервера придет в формате JSON-файла со следующей структурой:
```JSON
{
    "size_bytes": 2979
}
```
