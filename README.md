# Django Ads API

Проект представляет собой простое приложение на Django — REST API для сайта объявлений.
Реализовано без использования Django REST Framework (DRF), только встроенные возможности Django.

## Используемые технологии

- Python
- Django
- SQLite (по умолчанию)
- Pillow (для работы с изображениями)

## Установка и запуск

Клонировать репозиторий:

```bash
https://github.com/weahem/adservice
cd adservice
```

Создать виртуальное окружение:

```bash
python -m venv venv
```

Активировать виртуальное окружение:

```bash
venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Применить миграции:

```bash
python manage.py makemigrations
python manage.py migrate
```

Создать администратора:

```bash
python manage.py createsuperuser
```

Запустить сервер:

```bash
python manage.py runserver
```

Открыть приложение:

```
http://127.0.0.1:8000/
```

Админ-панель:

```
http://127.0.0.1:8000/admin/
```

Тестовая страница API:

```
http://127.0.0.1:8000/test_all_api.html
```

## API Эндпоинты

| Метод | URL | Описание | Авторизация |
|-------|-----|----------|-------------|
| GET | `/api/v1/trades/` | Список открытых объявлений (id, title, created) | Нет |
| POST | `/api/v1/trades/` | Создание объявления | Да |
| GET | `/api/v1/trades/{ID}/` | Полная информация об объявлении + изображения | Нет |
| PUT | `/api/v1/trades/{ID}/` | Обновление объявления (название, описание, статус) | Да |
| POST | `/api/v1/trades/{ID}/images/` | Добавление изображения к объявлению | Да |
| DELETE | `/api/v1/trades/{ID}/images/{IMAGE_ID}/` | Удаление изображения | Да |

## Структура данных

В базе данных используются три основные таблицы:

- **User** — кастомная модель пользователя с дополнительным полем `phone`;
- **Trade** — объявления, содержащие заголовок, текст, статус (открыто/закрыто), автора и даты создания/обновления;
- **Image** — изображения, привязанные к объявлениям, с указанием автора и дат создания/обновления.

Таблица `Trade` содержит:

- `title` — заголовок объявления;
- `text` — текст объявления;
- `status` — булевый флаг (True — открыто, False — закрыто);
- `author` — ссылка на пользователя;
- `created` и `updated` — даты создания и обновления.

Таблица `Image` содержит:

- `image` — файл изображения;
- `author` — ссылка на пользователя;
- `trade` — ссылка на объявление.

## Тестирование

Для удобного тестирования API в проекте предусмотрена HTML-страница с интерфейсом для всех эндпоинтов. После запуска сервера она доступна по адресу:

```
http://127.0.0.1:8000/test_all_api.html
```

На странице представлены кнопки для каждого метода API, поля для ввода данных и загрузки изображений. Результаты запросов отображаются в формате JSON.


## Особенности реализации

- **Кастомная модель пользователя** — добавлено поле `phone` к стандартной модели User.
- **Права доступа** — только автор объявления может его редактировать и добавлять/удалять изображения.
- **Валидация** — все ошибки возвращаются в формате JSON с соответствующими HTTP-статусами (400, 401, 403, 404).
- **Сериализация** — данные преобразуются в JSON без использования DRF, только встроенные средства Django (`JsonResponse`).
- **Изображения** — загружаются через `multipart/form-data` и сохраняются в папку `media/`.


## Регистрация и авторизация

На странице добавлена регистрация нового пользователя, вход и выход из учетной записи. По умолчанию стоят ограничения на пароль:
 - Длина не менее 8 символов
 - Отсутствие схожести с логином
 - Проверка на составленность только из цифр
 - Распространенность пароля

Вход в учетную запись производится через 

```
http://127.0.0.1:8000/login.html
```
Регистрация производится через страницу 

```
http://127.0.0.1:8000/register.html
```

Выход из учетной записи через страницу 

```
http://127.0.0.1:8000/logout.html
```
